## 1. 生产者-消费者问题

### 问题描述

- 一个或多个**生产者**生成数据放入**有界缓冲区**（容量 N）
- 一个或多个**消费者**从缓冲区取出数据
- 缓冲区满时生产者等待；缓冲区空时消费者等待
- 对缓冲区的访问必须**互斥**

### 信号量设计

| 信号量 | 初值 | 含义 |
|--------|------|------|
| `mutex` | 1 | 互斥访问缓冲区 |
| `empty` | N | 缓冲区空位数（生产者申请） |
| `full` | 0 | 缓冲区产品数（消费者申请） |

### 代码

```c
semaphore mutex = 1, empty = N, full = 0;

// Producer
while (1) {
    produce_item();
    P(empty);    // ① 先申请空位（资源信号量）
    P(mutex);    // ② 再申请互斥（互斥信号量）
    insert_item();
    V(mutex);    // ③ 释放互斥
    V(full);     // ④ 增加产品计数
}

// Consumer
while (1) {
    P(full);     // ① 先申请产品（资源信号量）
    P(mutex);    // ② 再申请互斥
    remove_item();
    V(mutex);    // ③ 释放互斥
    V(empty);    // ④ 增加空位计数
}
```

### 关键考点：P 操作顺序为什么不能颠倒？

按 `P(mutex)` → `P(empty)` 会发生什么：

```
1. 生产者拿到 mutex
2. 缓冲区满，P(empty) 阻塞
3. 消费者想消费 → P(mutex) 阻塞（生产者还拿着 mutex！）
4. 双方互相等待 → 死锁
```

> **铁律**：先申请**资源信号量**（有缓冲作用的），再申请**互斥信号量**。释放时顺序无所谓。

## 2. 读者-写者问题

### 问题描述

- **读者**：可同时读（共享）
- **写者**：写入时必须独占（互斥），且排斥读者
- 设计目标：最大化并发，同时保证数据一致

### 信号量设计

| 信号量 | 初值 | 含义 |
|--------|------|------|
| `rw_mutex` | 1 | 写者互斥 + 第一个读者锁写 |
| `mutex` | 1 | 保护 read_count 的互斥 |

### 读者优先版本

```c
semaphore rw_mutex = 1, mutex = 1;
int read_count = 0;

// Reader
P(mutex);
read_count++;
if (read_count == 1)
    P(rw_mutex);    // 第一个读者锁住写者
V(mutex);
// ... 读 ...
P(mutex);
read_count--;
if (read_count == 0)
    V(rw_mutex);    // 最后一个读者释放写者
V(mutex);

// Writer
P(rw_mutex);        // 排斥其他写者 + 等所有读者离开
// ... 写 ...
V(rw_mutex);
```

### 为什么是"读者优先"？

```
Reader1 在读 →
Writer 到达，P(rw_mutex) 阻塞 ←
Reader2 到达，read_count++（因为 mutex 不冲突）
→ Reader2 在读，Writer 继续等
→ 连续来读者，Writer 永远等不到 → 写者饥饿
```

读者可以"插队"——只要有一个读者在读，后来的读者不用等 rw_mutex（因为 read_count > 0 时不需要 P(rw_mutex)）。

### 写者优先思路

增加一个 `write_mutex` 信号量：写者到达时先 P(write_mutex)，后来的读者阻塞在 P(write_mutex)。写者结束后 V(write_mutex) 放行读者。

## 3. 哲学家就餐问题

### 问题描述

- 5 个哲学家围圆桌，5 根筷子（每两人之间一根）
- 吃饭需**同时拿左右两根**筷子
- 吃完放下，思考，循环

### 死锁场景

```
所有哲学家同时拿起左边的筷子 → 每人等右边 → 死锁
```

### 解法一：最多 4 人同时进食

```c
semaphore chopstick[5] = {1,1,1,1,1};
semaphore limit = 4;  // 限制同时拿筷子的人数

Philosopher(i):
    P(limit);               // 最多 4 人进入
    P(chopstick[i]);        // 拿左
    P(chopstick[(i+1)%5]);  // 拿右
    eat();
    V(chopstick[i]);
    V(chopstick[(i+1)%5]);
    V(limit);
```

> 4 人 + 5 根筷子 → 至少有一人能拿到两根（鸽巢原理）。

### 解法二：奇偶编号不同顺序

```c
Philosopher(i):
    if (i % 2 == 1) {
        P(chopstick[i]);        // 奇数：先左
        P(chopstick[(i+1)%5]);  // 后右
    } else {
        P(chopstick[(i+1)%5]);  // 偶数：先右
        P(chopstick[i]);        // 后左
    }
    eat();
    V(chopstick[i]);
    V(chopstick[(i+1)%5]);
```

> 相邻哲学家拿筷子顺序不同 → 不会形成环 → 无死锁。

### 解法三：AND 信号量（同时拿两根）

```c
Swait(chopstick[i], chopstick[(i+1)%5]);  // 同时申请两根
eat();
Ssignal(chopstick[i], chopstick[(i+1)%5]);
```

要么全拿，要么全不拿——破坏"持有并等待"条件。

## 4. 吸烟者问题

### 问题描述

- **3 个吸烟者**各有一种原料（烟草/纸/火柴），缺另外两种
- **1 个供应商**随机提供两种原料放在桌上
- 对应的吸烟者拿到原料后吸烟，完成后通知供应商继续

### 信号量设计

| 信号量 | 初值 | 含义 |
|--------|------|------|
| `agent` | 1 | 供应商可用 |
| `tobacco` | 0 | 有烟草（给缺烟草的吸烟者） |
| `paper` | 0 | 有纸 |
| `match` | 0 | 有火柴 |
| `done` | 0 | 吸烟者完成 |

这是单生产者-多消费者的变体——每次只唤醒**特定**的消费者（而非任意一个）。

## 5. 理发师问题

### 问题描述

- 理发店有 **1 个理发师** + **N 个等候椅**
- 无顾客时理发师**睡觉**
- 顾客到来：理发师在忙且有空椅 → 坐下等；无空椅 → 离开

### 信号量设计

| 信号量 | 初值 | 含义 |
|--------|------|------|
| `customers` | 0 | 等待理发的顾客数（理发师睡眠于此） |
| `barber` | 0 | 理发师就绪（顾客等待于此） |
| `mutex` | 1 | 保护 waiting 计数 |

```c
int waiting = 0;  // 当前等待顾客数
semaphore customers = 0, barber = 0, mutex = 1;

// Barber（循环）
while (1) {
    P(customers);       // 无顾客时睡眠
    P(mutex);
    waiting--;
    V(barber);          // 通知顾客可以理发了
    V(mutex);
    cut_hair();
}

// Customer
P(mutex);
if (waiting < N) {     // 有空椅
    waiting++;
    V(customers);       // 唤醒理发师（如果睡着）
    V(mutex);
    P(barber);          // 等待理发师就绪
    get_haircut();
} else {
    V(mutex);           // 无空椅，离开
}
```

> 核心：`customers` 和 `barber` 形成**两个同步点**——顾客发信号给理发师（有客人了），理发师发信号给顾客（可以理发了）。
