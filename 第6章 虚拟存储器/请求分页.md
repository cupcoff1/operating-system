# 请求分页

## 虚拟内存

每个进程有独立的**虚拟地址空间**，可大于物理内存。不常用的页换出到磁盘。

### 四大特征

| 特征 | 含义 |
|------|------|
| **离散性** | 进程装入内存不连续（分页/分段实现） |
| **多次性** | 程序分多次调入内存（区别于一次性装入） |
| **对换性** | 运行中进程的部分可换出/换入 |
| **虚拟性** | 逻辑上扩大内存，用户觉得内存很大 |

### 页面调入策略

| 策略 | 含义 |
|------|------|
| **预调页** | 首次装入时多调几页（猜测局部性），可能浪费 |
| **请调页** | 缺页时才调入（按需），现代 OS 标准 |

### 从何处调入

| 情况 | 来源 |
|------|------|
| 系统有足够对换区 | 进程装入时从文件区拷到对换区，之后从对换区换入换出 |
| 无对换区 | 不修改的从文件区直接调入，可能修改的写入对换区 |
| UNIX/Linux | 未修改页直接从文件区取（磁盘缓存），已修改页写对换区并记外存地址 |

## 缺页中断

```
访问页 P
  ├─ 在内存 → 正常访问 ✓
  └─ 不在内存（有效位=0）
      ├─ 合法地址 → 缺页中断：
      │    1. 在磁盘找到该页
      │    2. 找空闲帧（没有则页面置换）
      │    3. 读入内存
      │    4. 更新页表
      │    5. 重新执行缺页指令
      └─ 非法地址 → 段错误
```

## 有效访问时间

```
EAT = (1−p) × T_mem + p × T_page_fault

p = 缺页率（通常 0.001%~0.01%）
T_page_fault ≈ 10ms, T_mem ≈ 100ns
```

> 即使缺页率仅 0.001%，有效时间也会翻倍——磁盘 vs 内存差数十万倍。

## 页面分配策略

| 策略 | 含义 |
|------|------|
| **固定分配 + 局部置换** | 帧数固定，只换自己的页 |
| **可变分配 + 全局置换** | 动态调整帧数，可抢别人的帧 |
| **可变分配 + 局部置换** | 动态调整 + 只换自己的（最佳） |
