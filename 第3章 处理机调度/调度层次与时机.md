# 调度层次与时机

## 知识框架

```
处理机调度
├── 三级调度体系
│   ├── 高级调度（作业调度 / 长程调度）
│   │   └── 选后备作业进入内存，频率最低（分钟~小时）
│   ├── 中级调度（内存调度 / 中程调度）
│   │   └── 挂起/激活进程（换入/换出），频率中等
│   └── 低级调度（CPU 调度 / 短程调度）
│       └── 从就绪队列选进程分配 CPU，频率最高（ms 级）
├── 调度方式
│   ├── 非抢占式：运行到完成或阻塞才释放 CPU
│   └── 抢占式：更高优先级到达可强行剥夺 CPU
├── 调度时机
│   ├── 运行→阻塞（I/O 请求）
│   ├── 运行→就绪（时间片到）
│   ├── 阻塞→就绪（I/O 完成）
│   └── 运行→终止
└── 评价指标
    ├── 面向用户：周转时间、响应时间、截止时间
    └── 面向系统：CPU 利用率、吞吐量
```

---

## 1. 三级调度体系

### 为什么需要三级？

```
后备队列（磁盘）────高级调度───→ 就绪队列（内存）──低级调度───→ CPU
                   ↕ 中级调度
              挂起队列（磁盘对换区）
```

- **高级调度**：控制**并发度**——内存中放多少进程。太多则内存不足，太少则 CPU 空闲。
- **中级调度**：内存紧张时的"调压阀"——临时换出暂不运行的进程，避免 OOM。
- **低级调度**：每毫秒级的 CPU 分配决策，是调度器最频繁也最核心的工作。

### 三级对比

| 维度 | 高级调度 | 中级调度 | 低级调度 |
|------|----------|----------|----------|
| **别名** | 作业调度 / 长程调度 | 内存调度 / 中程调度 | CPU 调度 / 短程调度 / 进程调度 |
| **操作对象** | 后备作业 | 进程（挂起/激活） | 就绪进程 |
| **决策** | 选哪个作业进入内存 | 换出哪个 + 换入哪个 | 选哪个进程获得 CPU |
| **频率** | 最低（分钟~小时） | 中等（秒级） | **最高（毫秒级，每 10~100ms）** |
| **有无** | 批处理系统有，分时系统通常无 | 通用 OS 有 | 所有 OS 必有的核心 |
| **目标** | 控制并发度和资源分配 | 控制内存压力 | 最大化 CPU 利用率 + 公平响应 |

> 纯分时 OS（如早期 UNIX）没有高级调度——用户登录即创建进程，`fork()` 后直接进入内存竞争 CPU，只有低级调度。

## 2. 调度方式

### 非抢占式（Non-Preemptive）

进程自愿放弃 CPU 的三种情况：
1. 运行完成（终止）
2. 等待 I/O（阻塞）
3. 主动调用 `yield()` 放弃

> 简单，但一个死循环就能霸占 CPU。Windows 3.1 和早期 Mac OS 是非抢占式。

### 抢占式（Preemptive）

OS 可**强行**剥夺当前进程的 CPU：

| 触发条件 | 说明 |
|----------|------|
| 时间片耗尽 | RR 调度中定时器中断触发 |
| 更高优先级进程就绪 | 优先级调度中 I/O 完成唤醒 |
| 系统调用返回 | syscall 执行完返回用户态前检查 need_resched 标志 |

> 所有现代通用 OS（Linux/Windows/macOS）均为抢占式。

### 抢占点

操作系统在哪些时刻"拦截"并可能切换进程：

```
1. 系统调用返回时（用户态入口）
2. 中断返回时（内核态 → 用户态）
3. 当前进程在内核态阻塞时（如等待 I/O）
```

> 进程在内核态执行时，传统 Unix 不可抢占（可能长期占用）。Linux 2.6+ 引入**内核抢占**——即使在内核态，更高优先级任务也能抢占（需持有自旋锁时除外）。

---

## 3. 调度时机（触发低级调度的四种情况）

| 时机 | 触发 | 非抢占可以？ | 抢占可以？ |
|------|------|-------------|-----------|
| 运行→阻塞 | I/O 请求、`wait()` | ✓ | ✓ |
| 运行→就绪 | 时间片到、被抢占 | ✗ | ✓ |
| 阻塞→就绪 | I/O 完成、信号到达 | ✗ | ✓ |
| 运行→终止 | 进程 exit | ✓ | ✓ |

> 调度时机 ① 和 ④ 总是触发调度（CPU 必须重新分配）。时机 ② 和 ③ 只在**抢占式**下触发调度。

## 4. 调度评价指标

### 公式与含义

| 指标 | 公式 | 含义 |
|------|------|------|
| **CPU 利用率** | 有效 CPU 时间 / 总时间 | 有多忙 |
| **吞吐量** | 单位时间完成的作业数 | 做得多快 |
| **周转时间** | 完成时刻 − 到达时刻 | 等了多久 + 做了多久 |
| **带权周转时间** | 周转时间 / 运行时间 | 等待代价（越小越好，理想值 = 1） |
| **等待时间** | 在就绪队列的总时间 | 纯等待，不含执行 |
| **响应时间** | 首次响应 − 请求时刻 | 交互体验（分时系统最关心的指标） |

### 关系

```
周转时间 = 等待时间 + 运行时间 + I/O 时间
带权周转时间 = 1 + 等待时间/运行时间  （纯计算时忽略 I/O）
```

### 例子

| 进程 | 到达 | 运行 | 完成 | 周转 | 带权周转 | 等待 |
|------|------|------|------|------|----------|------|
| A | 0 | 10 | 10 | 10 | 1.0 | 0 |
| B | 3 | 5 | 15 | 12 | 2.4 | 7 |

### 不同调度算法的目标侧重

| 系统类型 | 最关心指标 | 原因 |
|----------|-----------|------|
| **批处理** | 吞吐量、CPU 利用率 | 无用户交互，只管效率 |
| **分时** | 响应时间 | 用户敲键后等多久看到反应 |
| **实时** | 截止时间满足率 | 超过 deadline 可能灾难性 | 实时 | 截止时间满足率 |
